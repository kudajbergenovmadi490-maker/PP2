# Practice 5 — Python Regular Expressions (RegEx)

## Overview
This practice covers Python's `re` module through a hands-on receipt parsing exercise.

## Files
| File | Description |
|---|---|
| `receipt_parser.py` | Main script — parses `raw.txt` using regex |
| `raw.txt` | Sample supermarket receipt in plain text |
| `parsed_receipt.json` | Output: structured JSON (generated by the script) |
| `README.md` | This file |

## How to Run
```bash
python receipt_parser.py
```
> Requires Python 3.10+ (uses `float | None` union type hint).

## What the Script Demonstrates

### `re` Functions Covered
| Function | Used For |
|---|---|
| `re.findall()` | Extract all prices, quantities, product names |
| `re.search()` | Find date, time, totals, payment info |
| `re.split()` | Split receipt into sections; tokenise lines |
| `re.sub()` | Redact card number; normalise price format; clean text |
| `re.match()` | Check whether a string starts with a pattern |

### Flags
- `re.IGNORECASE` — case-insensitive label matching
- `re.MULTILINE` — `^` / `$` anchors match per line

### Concepts
- Metacharacters: `. * + ? ^ $ [] | () \`
- Special sequences: `\d \w \s \D \W \S \b`
- Quantifiers: `{n}`, `{n,}`, `{n,m}`
- Capturing groups `()`
- Character classes `[A-Z]`
